# Code of Conduct

## Our Commitment
KlyvantaDocs exists to help teams ship clear, reliable documentation fast.
We want this community to be welcoming, professional, and constructive for maintainers, contributors, and users from all backgrounds.

## Expected Behavior
- Be respectful and assume positive intent.
- Keep feedback specific, actionable, and product-focused.
- Share context when proposing changes, especially around docs UX, SEO, and authoring quality.
- Help others learn by explaining trade-offs, not just outcomes.
- Respect maintainers' time by following templates and contribution guidelines.

## Unacceptable Behavior
- Harassment, hate speech, discrimination, or personal attacks.
- Sexualized language or imagery.
- Doxxing, threats, intimidation, or stalking.
- Trolling, deliberate disruption, or bad-faith arguments.
- Publishing private information without explicit permission.

## Scope
This Code of Conduct applies in:
- GitHub issues, pull requests, discussions, and reviews
- project documentation and examples
- any community space where someone is representing this project

## Reporting
If you experience or witness behavior that violates this Code of Conduct:
1. Contact maintainers through GitHub contact options on the maintainer profile.
2. If a private channel is unavailable, open an issue titled `[Code of Conduct]` with only the minimum details needed to start review.
3. For safety-sensitive situations, prioritize your local authorities and personal safety first.

All reports will be reviewed as quickly as possible and handled respectfully.

## Enforcement
Maintainers may take any action they deem appropriate, including:
- private warning
- temporary suspension from project interactions
- permanent ban from project spaces

## Project Values
When decisions are unclear, we optimize for:
- clarity over complexity
- user trust over shortcuts
- long-term maintainability over short-term speed
